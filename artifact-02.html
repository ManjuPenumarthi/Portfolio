Machine Learning vs. Deep Learning: A Case Study Comparison

ARTIFACT 02

To build real intuition for when to use machine learning versus deep learning, I moved past textbook definitions and grounded the comparison in two real, deployed systems: Zillow's Zestimate (ML) and Oracle NetSuite's Bill Capture (DL). This artifact captures the reasoning I used to determine not just what each approach does, but why it's the right — or wrong — tool for a given kind of data.

Description A comparative case study analyzing one machine learning application (Zillow's Zestimate home-valuation model) and one deep learning application (Oracle NetSuite's Bill Capture, a CNN-based OCR system for invoice processing). For each, I identified the real-world system, explained why its approach fits the problem, and argued why the alternative approach would fail or underperform.
Objective To move beyond definitions and demonstrate practical judgment: given a real business problem and its data, correctly reason through whether classic ML or deep learning is the appropriate approach.
Process I researched how each system actually works, then built the argument in both directions — why structured, tabular data (square footage, comps, tax records) is well suited to classic ML and poorly suited to DL's added complexity; and why raw, highly variable image data (scanned invoices with no consistent template) requires DL's automatic feature learning and would break under a hand-coded ML approach.
Tools & Tech Machine Learning (regression, tree-based models), Deep Learning (CNNs, OCR), Applied AI/ML Case Analysis
Value Proposition This shows I can translate a core AI/ML distinction into a decision-making framework — the kind of reasoning that matters when scoping whether a project needs a lightweight ML model or a heavier DL system, rather than defaulting to whichever is trendiest.
Unique Value I intentionally chose Bill Capture as the DL example because it lives in the ERP/finance-automation space I already work in at Corey Construction — connecting the coursework directly to the kind of automated document processing my own industry is moving toward.
Relevance As I bring more structured AI evaluation into my NetSuite work, this is the exact judgment call I'll face: when a business problem calls for a simple, interpretable model versus when it genuinely needs deep learning's pattern-extraction power.
